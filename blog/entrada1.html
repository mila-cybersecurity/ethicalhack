Todo sobre IDORs: Insecure Direct Object Reference

Las Referencias Directas Inseguras a Objetos (IDOR) son una vulnerabilidad crítica en el ámbito de la seguridad informática. Ocurren cuando una aplicación web permite a los usuarios acceder a objetos directamente mediante parámetros que pueden ser manipulados. En este artículo, exploraremos en profundidad qué son las IDORs, su historia, impacto, cómo los ciberdelincuentes pueden explotarlas, ejemplos prácticos, técnicas para encontrarlas y medidas de mitigación que se pueden implementar para asegurar las aplicaciones web.

Ejemplos de IDOR
Acceso a Cuentas de Clientes:

Imagina un sitio web donde los usuarios pueden acceder a sus cuentas mediante la siguiente URL:

https://example.com/account?user_id=1001

Si un atacante cambia el parámetro user_id a 1002, podría acceder a la cuenta de otro usuario sin tener permiso.

Descarga de Archivos:

Considere una URL que permite a los usuarios descargar archivos de su cuenta:

https://example.com/download?file_id=1234

Un atacante podría modificar file_id a otro número para descargar archivos que no le pertenecen.

Información de Pedido:

En una tienda en línea, un usuario puede ver su historial de pedidos con la siguiente URL:

https://example.com/orders?order_id=5678

Si no hay controles adecuados, un atacante podría acceder a los pedidos de otros usuarios simplemente cambiando el order_id.

Acceso a Mensajes Privados:

Una aplicación de mensajería que almacena mensajes con un ID en la URL:

https://example.com/messages?msg_id=4321

Un atacante podría manipular el msg_id para acceder a los mensajes de otros usuarios.
