Todo sobre IDORs: Insecure Direct Object Reference

Las Referencias Directas Inseguras a Objetos (IDOR) son una vulnerabilidad crítica en el ámbito de la seguridad informática. Ocurren cuando una aplicación web permite a los usuarios acceder a objetos directamente mediante parámetros que pueden ser manipulados. En este artículo, exploraremos en profundidad qué son las IDORs, su historia, impacto, cómo los ciberdelincuentes pueden explotarlas, ejemplos prácticos, técnicas para encontrarlas y medidas de mitigación que se pueden implementar para asegurar las aplicaciones web.

¿Qué son las IDOR?

Las IDORs se producen cuando una aplicación web utiliza identificadores de objetos proporcionados por el usuario sin realizar una adecuada validación de permisos. Esto permite a los atacantes modificar los parámetros en las solicitudes para acceder a recursos no autorizados. Esta vulnerabilidad es un tipo de control de acceso inadecuado y puede llevar a la exposición de datos sensibles o a la realización de acciones no autorizadas en nombre de otros usuarios.

Historia y Descubrimiento

El término IDOR comenzó a ganar popularidad a mediados de la década de 2000, especialmente después de su inclusión en el Top Ten de OWASP en 2007. A medida que las aplicaciones web comenzaron a utilizar más identificadores en las URL y los formularios, se hicieron evidentes los riesgos asociados con la falta de controles de acceso adecuados. Aunque ha habido esfuerzos para abordar esta vulnerabilidad, sigue siendo un problema común en muchas aplicaciones web.

Impacto de las IDOR

Las IDORs pueden tener repercusiones graves tanto para los usuarios como para las organizaciones. Algunas de las consecuencias más comunes incluyen:

  • Exposición de Datos Sensibles: Los atacantes pueden acceder a información personal identificable (PII), registros médicos, información financiera y más.
  • Pérdida de Confianza: Si un usuario se entera de que su información ha sido comprometida, puede perder la confianza en la organización, lo que puede afectar la reputación de la empresa.
  • Consecuencias Legales: Las violaciones de datos pueden llevar a acciones legales y multas significativas, especialmente si se trata de información regulada.
Ejemplos de IDOR
  1. Acceso a Cuentas de Clientes:

    Imagina un sitio web donde los usuarios pueden acceder a sus cuentas mediante la siguiente URL:

    https://example.com/account?user_id=1001

    Si un atacante cambia el parámetro user_id a 1002, podría acceder a la cuenta de otro usuario sin tener permiso.

  2. Descarga de Archivos:

    Considere una URL que permite a los usuarios descargar archivos de su cuenta:

    https://example.com/download?file_id=1234

    Un atacante podría modificar file_id a otro número para descargar archivos que no le pertenecen.

  3. Información de Pedido:

    En una tienda en línea, un usuario puede ver su historial de pedidos con la siguiente URL:

    https://example.com/orders?order_id=5678

    Si no hay controles adecuados, un atacante podría acceder a los pedidos de otros usuarios simplemente cambiando el order_id.

  4. Acceso a Mensajes Privados:

    Una aplicación de mensajería que almacena mensajes con un ID en la URL:

    https://example.com/messages?msg_id=4321

    Un atacante podría manipular el msg_id para acceder a los mensajes de otros usuarios.

Google Dorks para Identificar IDORs

Los Google Dorks son comandos de búsqueda que ayudan a los investigadores de seguridad a identificar posibles IDORs en aplicaciones web. Aquí hay algunos ejemplos:

  • Buscar parámetros de usuario: 
    site:example.com inurl:user_id
  • Identificar archivos expuestos: 
    site:example.com ext:pdf OR ext:txt

Estos comandos permiten a los investigadores encontrar rápidamente posibles puntos débiles en la seguridad de una aplicación.

Tipos de IDOR
  • Escalación de Privilegios Horizontal: Cuando un usuario accede a recursos de otro usuario en el mismo nivel de privilegio.
  • Escalación de Privilegios Vertical: Ocurre cuando un usuario sin privilegios accede a recursos que están restringidos a usuarios con mayores privilegios.
Cómo los Ciberdelincuentes Pueden Utilizar IDOR

Los ciberdelincuentes explotan IDORs manipulando los parámetros en las solicitudes HTTP. Algunas técnicas incluyen:

  • Alteración de Parámetros: Modifican directamente los valores en las URL para acceder a objetos no autorizados.
  • Automatización de Ataques: Utilizan herramientas para realizar ataques de fuerza bruta, probando diferentes valores para encontrar objetos accesibles.
  • Ingeniería Social: A veces, los atacantes obtienen información sobre otros usuarios y utilizan IDORs para acceder a sus cuentas.
Formas de Mitigar IDORs

Para protegerse contra IDORs, las organizaciones deben implementar las siguientes medidas:

  • Validación de Entrada: Validar todos los parámetros de entrada para asegurarse de que los usuarios tienen permiso para acceder a los recursos solicitados.
  • Implementación de Controles de Acceso: Asegurarse de que las aplicaciones implementen controles de acceso robustos y verifiquen que el usuario tiene los permisos adecuados.
  • Uso de Identificadores Aleatorios: Emplear identificadores únicos (UUID) o aleatorios en lugar de números secuenciales predecibles.
  • Monitoreo y Registro: Mantener registros de acceso y monitorear las actividades para detectar comportamientos sospechosos.
  • Pruebas de Seguridad: Realizar pruebas de penetración periódicas para identificar y corregir vulnerabilidades de IDOR.
Conclusión

Las Referencias Directas Inseguras a Objetos (IDOR) representan una amenaza significativa para la seguridad de las aplicaciones web. La falta de controles de acceso adecuados puede llevar a la exposición de datos sensibles y a la pérdida de confianza por parte de los usuarios. Al entender cómo funcionan las IDORs y al implementar medidas de mitigación adecuadas, las organizaciones pueden protegerse contra estas vulnerabilidades.
